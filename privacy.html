隐私政策

最后更新：2026年5月10日

应用名称：MyComposioTikTokApp

开发者：Hui Zhou（阿辉）

1. 我们收集的信息

本应用 MyComposioTikTokApp 旨在通过 TikTok 官方 API 为用户提供内容发布、自动化管理与集成服务。我们不会主动收集、存储或上传您的任何个人身份信息（如姓名、地址、电话号码、财务信息等）。

在您使用 TikTok OAuth 2.0 授权登录时，本应用仅通过 API 获取以下有限权限（具体取决于您授权的范围）：

  • 公开信息：您 TikTok 账户的公开资料（如昵称、头像、粉丝数等）。
  • 内容发布：在您的授权下，代表您发布视频、图文等内容（仅通过 API 操作，本应用不保存内容副本）。
  • 数据洞察：如果您授权了数据权限，本应用可获取公开视频的分析数据（用于研究或自动化报告）。

所有数据均直接通过 TikTok 官方服务器传输，我们不存储任何访问令牌或刷新令牌，OAuth 流程完全由 Composio 平台托管（Composio 是行业标准的第三方集成平台，其安全策略请参考 Composio 官方文档）。

2. 信息的使用方式

我们仅使用获取的 API 数据来完成您实时请求的操作，例如：

  • 根据您的指令自动发布内容；
  • 查询并展示您的 TikTok 公开统计数据；
  • 连接其他自动化工具（如 Composio）以构建工作流。

我们不会将您的数据用于任何训练、转售、广告定向或与原始请求无关的用途。

2. 信息共享与披露

我们不会与任何第三方共享您的个人数据，除非：

  • 获得您的明确同意；
  • 法律要求（如法院命令、传票）；
  • 为保护 TikTok、用户或公众的权益而必要。

4. 数据保留

由于我们不存储任何用户数据，您的信息仅在当前 API 请求会话期间临时存在于内存中，请求结束后即被完全销毁。TikTok 官方会根据其自身策略保留您的账户数据，详情请参阅 TikTok 隐私政策。

5. 您的权利

您可以随时：

  • 通过 TikTok 开发者平台的 App 管理页面 撤销对本应用的授权；
  • 联系我（邮箱见下方）要求删除我们可能意外保留的任何残留信息（尽管我们设计为无状态）。

6. 托管说明

本隐私政策页面由 GitHub Pages 托管。当您访问此页面时，GitHub 可能会根据其隐私政策收集您的 IP 地址、浏览器类型和访问时间等标准日志信息。GitHub 的数据处理方式请参阅 GitHub 隐私声明。

7. 政策更新

我们可能会不定期更新此隐私政策。重大变更将在本页面显著位置公告，并通过邮件（如果您已订阅）或通过 Composio 平台通知您。变更自发布之日起生效。
